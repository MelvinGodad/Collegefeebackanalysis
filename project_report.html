Campus Life Analytics

Comprehensive Technical Report & System Documentation

Prepared By: Melvin Godad

1. Executive Summary

The Campus Life Analytics project is a data-driven initiative designed to modernize the way educational institutions collect, analyze, and act upon student feedback. Traditionally, event feedback is collected via scattered Google Forms and analyzed manually, leading to delays and lost insights.

This system automates the entire pipeline: from data simulation and cleaning to sentiment analysis using Natural Language Processing (NLP) and dynamic visualization. The result is a real-time, interactive dashboard that empowers campus organizers to make informed decisions to improve student satisfaction and engagement.

2. Project Objectives

  • Data Centralization: Aggregate feedback from multiple events into a unified structure.
  • Automated Insights: Use algorithms to calculate satisfaction scores and classify feedback topics automatically.
  • Dynamic Reporting: Replace static PDFs with an interactive web dashboard.
  • Scalability: Built on a modular Python and JavaScript architecture to handle extensive datasets.

3. Technology Stack & Libraries

This project leverages a robust stack of open-source tools to ensure reliability, performance, and ease of maintenance.

3.1 Backend & Data Processing (Python)

  • Python 3.x: The core programming language used for scripting the data generation and analysis pipelines availability, and rich ecosystem of data libraries.
  • pandas (`import pandas as pd`): Used for high-performance data manipulation. It handles CSV loading, cleaning (removing nulls), and aggregations (grouping by Event/Department).
  • TextBlob (`from textblob import TextBlob`): A simplified NLP library used for Sentiment Analysis. It calculates a 'Polarity' score (-1 to +1) for each comment to determine if feedback is Positive, Neutral, or Negative.
  • Comparison to Alternatives: TextBlob was chosen over VADER or NLTK for its lightweight nature and sufficient accuracy for general feedback.
  • os / json: Standard libraries used for file system operations and exporting processed data to the frontend in JSON format.

3.2 Frontend & Visualization (Web)

  • HTML5: Provides the semantic structure for the Landing Page and Dashboard.
  • CSS3 (Modern): Utilizes CSS Variables, Flexbox, and CSS Grid for a responsive, "glassmorphism" aesthetic (Dark Mode, gradients, blurred cards).
  • JavaScript (ES6+): Handles client-side logic, including navigation (SPA-like tabs), filters, and data fetching.
  • Chart.js: A powerful library for rendering interactive canvas-based charts. Used for:
    • Bar Charts (Ratings Distribution)
    • Doughnut Charts (Sentiment Breakdown)
    • Horizontal Bar Charts (Event Rankings)

4. System Architecture

The system is divided into three distinct modules: Data Generation, Analysis Engine, and Presentation Layer.

4.1 Module 1: Data Generation

Script: scripts/generate_feedback.py

Since real data is often sensitive or unavailable during development, a custom simulation engine was built. It generates realistic events ("Tech Fest", "Cultural Night") and correlates ratings with comments (e.g., a "1-star" rating is paired with negative text like "Bad experience"). 

# Example Generation Logic
data = {
    "Event": random.choice(events),
    "Rating": random.choices([1,2,3,4,5], weights=[5,5,15,40,35])[0],
    "Comment": correlate_comment(rating)
}

4.2 Module 2: Analysis Engine

Script: scripts/analyze_feedback.py

This is the brain of the system. It performs:

  1. Ingestion: Reads `student_feedback.csv`.
  2. NLP Processing: Computes Sentiment Polarity.
  3. Classification: Keyword-based categorization (e.g., words like "speaker", "sound" -> "Venue & Logistics").
  4. Export: Aggregates metrics and writes `dashboard_data.js` for the frontend to consume directly.

5. Dashboard Features

5.1 Interactive Filters

The dashboard includes a real-time Department filter. When a user selects "IT" or "Management", the JavaScript engine re-aggregates the raw data on the client side and instantly updates all charts. This allows for granular analysis without page reloads.

5.2 Tabbed Navigation

To keep the interface clean, content is organized into logical views:

  • Dashboard: High-level KPIs and Charts.
  • Analytics: Raw tabular data with search and full comment viewing.
  • Events: Scorecards for individual event performance.
  • Settings: System status and controls.

5.3 Landing Page

To present a professional face to the project, a landing page (`index.html`) was designed. It features a hero section, feature highlights, and direct links to the tools, serving as the entry point for stakeholders.

6. Conclusion & Future Scope

The Campus Life Analytics project successfully demonstrates how modern web technologies and Python-based data science can converge to solve real administrative problems. By automating the feedback loop, the system saves hundreds of hours of manual compilation and provides instant visibility into student sentiment.

Future Enhancements

  • Database Integration: Move from CSV to SQLite/PostgreSQL for persistent storage.
  • Authentication: Add Login for Admin vs. Student views.
  • Real-time API: Replace static JSON export with a Flask/FastAPI backend for live updates.
  • Advanced NLP: Implement Topic Modeling (LDA) for deeper insight into unstructured text.
